# 90 Days DevOps Learning Plan

## Current Level

I am a junior full-stack developer currently working as a Project Coordinator and Data Analyst while preparing to transition into Software Engineering, DevOps, and Cloud Engineering.

I already have experience with Python, Flask, FastAPI, JavaScript, SQL, Git, GitHub, Linux through Codespaces, REST APIs, and basic Docker. I have also built portfolio projects such as SaaS APIs, Shopify apps, dashboards, and automation tools.

## My Understanding of DevOps and Cloud

DevOps is the practice of delivering software reliably through automation, collaboration, monitoring, and continuous improvement. Cloud Engineering provides the infrastructure and services needed to run applications securely, reliably, and at scale.

My goal is to own the full lifecycle of modern software: development, deployment, monitoring, troubleshooting, and continuous improvement.

## Why I Am Learning DevOps

I am learning DevOps because I want to become a Software Engineer who can build production-ready systems, not only local projects. These skills will help me deploy my own SaaS products, automate delivery, improve reliability, and become stronger for AI-focused companies or high-growth startups.

## My 90-Day Goals

1. Build and document a complete CI/CD pipeline using GitHub Actions.
2. Deploy containerized applications using Docker, Kubernetes, and cloud infrastructure.
3. Create a DevOps-focused portfolio project that demonstrates real production practices.

## Core Skills I Want to Develop

- Linux administration and troubleshooting
- CI/CD automation with GitHub Actions
- Kubernetes, cloud infrastructure, and deployment operations

## Weekly Time Commitment

- Monday to Friday: 2 hours per day
- Saturday: 4 hours
- Sunday: 4 hours

Total: approximately 18 hours per week.

## Consistency Plan

I will study every day, even if some days are only one focused hour. For each topic, I will understand the theory, practice the commands, build something real, document my work on GitHub, and share progress publicly on LinkedIn.

My objective is consistency over perfection. After these 90 days, I want to confidently deploy, automate, troubleshoot, and manage modern cloud-native applications.
